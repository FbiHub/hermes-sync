# Skill: user-md-optimization

This skill manages a central `/home/fabianogori/user.md` file that serves as a high-level context document for our collaboration. It reduces the need for me to re-learn your preferences or project details.

## How to use
1. **Initial Setup:** I will create/update the `user.md` file based on our conversations.
2. **Maintenance:** Whenever you share a preference, project constraint, or working habit, I will update this file.
3. **Usage:** Before tackling complex tasks, I check this file to ensure I am aligned with your expectations.

## Structure of user.md
- **Personal Preferences:** (Communication style, formatting preferences)
- **Project Context:** (Repositories, key workflows, tech stack)
- **Development Standards:** (Testing rules, commit messages, code quality)
- **Environment:** (Tool paths, configuration quirks)

## Maintenance Routine
- Periodically summarize lessons learned or refined workflows into this file.
- If we change a major workflow, update this file to reflect the new "source of truth."
